SECTION 1 — CORE PRINCIPLES
====================================================================

1. Database is the ONLY source of truth.
2. FSM is NEVER source of truth — only UI state guide.
3. Every critical operation must be restart-safe.
4. Every critical operation must be idempotent.
5. Every critical operation must be audit-logged.
6. Every critical operation must be recoverable.
7. No critical state may depend solely on memory or FSM.
8. All critical flows must be async-safe, duplicate-safe,
   and FloodWait-safe.
9. One Telegram User = One Permanent Topic.
10. All user activity must be routed into the user topic.

--- 4.1 BOT COMMAND MENU (Telegram Slash Menu) ---

  /start     →  Main Menu
  /takedown  →  Remove Content
  /help      →  Support

--- 4.2 COMMAND BEHAVIOR ---

  /start
    → Opens main menu.
    → First-time users see onboarding first, then main menu.

  /takedown
    → Opens Takedown Request flow.
    → Full flow defined in Section 14.

    → Creates or reopens support session in User Topic.
    → Full flow defined in Section 15.

--- 4.3 DOT-SLASH PREFIX RULE ---

  Any message or command starting with  ./  (dot-slash prefix):
    • Deleted automatically after 10 seconds.
    • Silent deletion only.

SECTION 7 — PREMIUM SUBSCRIPTION SYSTEM
====================================================================

--- 7.1 PACKAGES ---
1 Month | 3 Months | 6 Months (Configurable)

--- 7.2 SUBSCRIPTION MODEL ---
Statuses: ACTIVE | EXPIRED | CANCELLED | PENDING

--- 7.3 FULL PAYMENT FLOW ---
User
  → Clicks Premium Access
  → Selects Package/Method
  → Payment Request Created
  → Route To User Topic (Hub)
  → Admin Reviews Request
  → Admin Manually Sends Details
  → Bot confirms delivery success
  → Payment Session Activated (20 minutes)
  → User Submits Proof (TXID + optional Screenshot)
  → Proof routed to User Topic
  → Admins See:  ✅ Approve  |  ❌ Reject
  → Admin Approves → Subscription Activation Flow

CRITICAL RULE: Details ALWAYS sent manually by admin.

SECTION 8 — USER TOPIC SYSTEM (USER-CENTRIC)
====================================================================

RULE: One User = One Permanent Topic.

1. Every user receives exactly one topic in the Hub.
2. The topic is reused forever.
3. Mapping (user_id ↔ topic_id) is persisted in MongoDB (Source of Truth).
4. Format: 👤 Full Name | User ID
5. All user activity (Support, Payments, Submissions, Moderation) is routed here.

SECTION 11 — CONTENT SUBMISSION SYSTEM
====================================================================

User Submits Content
  → Content routed to User Topic
  → Admins Review
  → Actions: 🔞 Approve NSFW | ⭐ Approve Premium | ❌ Reject (mandatory reason)

On Approval: Vault Archive → Queue job → Scheduled Posting.
On Rejection: Status updated in User Topic → Reason stored → User notified.

====================================================================
SECTION 12 — VAULT SYSTEM
====================================================================

Permanent archive of all approved content.
Two separate vaults: NSFW Vault and Premium Vault.
Supports random reposting and fairness rotation.

====================================================================
SECTION 13 — QUEUE DISTRIBUTION ENGINE
====================================================================

PENDING → PROCESSING → SENT | FAILED → DEAD_LETTER
Pipeline includes Fairness Layer, Rate Limiter, and FloodWait Handler.

====================================================================
SECTION 14 — TAKEDOWN REQUEST SYSTEM
====================================================================

/takedown → Collect Reason → Collect Link.
Request routed to User Topic in Hub.
Admin: ✅ Approve & Delete | ❌ Reject (prompts for reason).
On Rejection: User notified + Support session auto-opened in same User Topic.

SECTION 15 — SUPPORT SYSTEM
====================================================================

Entry: Need Help Button | /help | Unhandled Messages.
Flow:
User Message → User Topic → Support Status = PENDING
Admin → /accept → Support Status = ACTIVE (Two-way bridge opens)
Admin → /close → Support Status = CLOSED (User side deleted)
Topic remains permanently with full history.
