WORKFLOW ISOLATION RULES
━━━━━━━━━━━━━━━━━━━━━━

The following systems MUST remain isolated:

A) Payment System
B) Support System
C) Moderation System
D) Distribution System

NEVER ALLOW:

* payment proofs into support
* moderation callbacks into payment
* support interception of payment flows
* subscription logic inside moderation queue

FSM SAFETY RULES
━━━━━━━━━━━━━━━━━━━━━━

FSM uses:

* aiogram FSM
* MemoryStorage

MemoryStorage is NOT trusted persistence.

Database is source of truth.

FSM may ONLY control:

* temporary interaction state
* current conversation step

Critical workflow state MUST persist in database.
