            await asyncio.sleep(e.value + 1)
            retry once
          except (UserBlocked, UserDeactivated, ChatNotFound):
            log failed, continue to next user

      Update broadcast_job: sent_count, failed_count, status = DONE
      Tell admin in topic:
          "📢 Broadcast complete.
           Sent: [count] | Failed: [count]"

FLOW K: VAULT & QUEUE ENGINE
--------------------------------------------------------------------

WHAT MUST HAPPEN:

  K1. Queue worker (runs continuously, asyncio loop):
        Every N minutes (configurable via ENV: QUEUE_INTERVAL_MINUTES):
          Query queue_jobs where status = PENDING, ordered by scheduled_at
          Check today's sent count per category vs daily limits:
            PREMIUM_DAILY_LIMIT (from ENV)
            NSFW_DAILY_LIMIT    (from ENV)
          For each eligible job:
            Update status = PROCESSING
            Apply watermark (see Flow L)
            Send to target group(s)
            If success: status = SENT, processed_at = now()
            If fail: retry_count += 1
              If retry_count >= max_retries: status = DEAD_LETTER
              Else: status = PENDING (will retry next cycle)

  K2. Vault fallback (when queue is empty):
        If no PENDING jobs exist:
          Query vault where:
            last_posted_at < now() - VAULT_COOLDOWN_HOURS (from ENV)
            OR last_posted_at IS NULL
          Order RANDOMLY (ORDER BY RANDOM() in SQL)
          Pick one item per category up to daily limits
          Create queue_job for it
          Proceed to K1

  K3. Dead letter queue:
        Admin can send /deadletters in hub to see dead letter jobs
        Admin can manually retry or dismiss them
