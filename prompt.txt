## STEP 1 — BUGS TO FIX (fix all, no exceptions)

### CRITICAL (P0)

**B-01 · Referral TTL index + partialFilterExpression**
In `app/referral/repository.py`, the TTL index combines `expireAfterSeconds`
with `partialFilterExpression`. This fails on MongoDB < 5.0 and even on 5.0+
the enum object (not `.value`) is passed as the filter value, so it never
matches. Remove the TTL+partial combination entirely. Replace with a
`purge_stale_pending(hours=48)` method called from the qualification job.

**B-02 · QueueRepository missing `db` argument**
In `app/handlers/user_handler.py`, `_get_queue_repo()` calls `QueueRepository()`
with no arguments. `QueueRepository.__init__` requires `db: AsyncIOMotorDatabase`.
Fix: `return QueueRepository(DatabaseManager.get_db())`.

**B-03 · Forum topic creation uses non-existent API**
In `app/payments/topics.py`, `client.create_forum_topic(...)` does not exist in
pyrofork. Fix: use raw API `client.invoke(raw.functions.channels.CreateForumTopic(...))`
exactly as `app/services/topic_service.py` does.

**B-04 · Double `callback.answer()` in payment handlers**
`handle_premium_menu`, `handle_plan_selection`, `handle_payment_method` all
call `await callback.answer()` twice. Remove every duplicate call. Rule: one
`callback.answer()` per handler, called immediately at the top.

**B-05 · `non_core_index_setup_failed` silently swallows actual error**
In `app/core/database.py`, the referral and payment index creation errors are
caught but the `exc_info=True` may still mask the real message. Ensure every
index error logs `repr(e)` and the collection name.

**B-06 · `handle_payment_inputs` races with `handle_media_submission`**
Both are `filters.private & ~filters.regex(r"^/")`. `handle_payment_inputs`
correctly raises `ContinuePropagation` when no session exists, but it still
makes a MongoDB call (`get_active_session`) for EVERY private message from
EVERY user. Add a Redis-backed fast check: if no `pay_session:{user_id}` key
exists in Redis, raise `ContinuePropagation` immediately without a DB call.
Set the Redis key when a session is created; clear it on approval/rejection/expiry.

**B-07 · `approve_payment` loads stale session before lock**
After `acquire_processing_lock` changes DB status to `PROCESSING`, the local
`session` object still has `status=UNDER_REVIEW`. This is safe for the happy
path but causes incorrect rollback logging on failure. After acquiring the lock,
reload: `session = await self.repository.get_session(payment_id)`.

**B-08 · Support handler routes ALL private text to DB even with no topic**
`handle_private_message_support` calls `get_user_topic_id` for every private
non-command text message. Add Redis cache: `support_topic:{user_id}` — set on
topic creation, cleared on close. Check Redis first; only hit MongoDB on miss.

### F-08 · Dual Watermark System (photos AND videos)

#### Photos — Dual PNG Watermark
Apply the watermark PNG image TWICE at two different random positions.

```python
def _random_position_expr(img_w, img_h, wm_w, wm_h, margin=20):
    """Return a random (x, y) that keeps the watermark inside the image."""
    x = random.randint(margin, max(margin + 1, img_w - wm_w - margin))
    y = random.randint(margin, max(margin + 1, img_h - wm_h - margin))
    return x, y
```

FFmpeg filter for photos (two overlays at two independent random positions):
```
[1:v]scale=iw*{scale}:-1,format=rgba,colorchannelmixer=aa={opacity}[wm1];
[1:v]scale=iw*{scale2}:-1,format=rgba,colorchannelmixer=aa={opacity2}[wm2];
[0:v][wm1]overlay={x1}:{y1}[tmp];
[tmp][wm2]overlay={x2}:{y2}[out]
```

- `scale` and `scale2`: independently random between 0.10 and 0.20
- `opacity` and `opacity2`: between 0.6 and 0.9
- `(x1,y1)` and `(x2,y2)`: generated randomly with no positional constraint
  (they can overlap — that is intentional for watermark protection)
- The second watermark should use a DIFFERENT random scale than the first

#### Videos — Dual Text Watermark
Apply the watermark text TWICE using the `drawtext` ffmpeg filter.

Each text watermark has:
- Independent random `(x, y)` position
- Independent random start time within the video duration
- Independent random opacity (0.4–0.8)
- Font path from `WATERMARK_FONT_PATH` env var (fallback to system font)
- Text value from `WATERMARK_TEXT_NSFW` or `WATERMARK_TEXT_PREMIUM`

FFmpeg filter for videos:
```
drawtext=text='{text}':x={x1}:y={y1}:fontsize=24:fontcolor=white@{a1}:
fontfile={font}:enable='between(t,{t1},{end1})',
drawtext=text='{text}':x={x2}:y={y2}:fontsize=28:fontcolor=white@{a2}:
fontfile={font}:enable='between(t,{t2},{end2})'
```

Where:
- `t1`, `t2`: random start times within video duration (use ffprobe to get duration)
- `end1`, `end2`: `t1 + random(3, 8)` seconds
- `x1`, `x2`, `y1`, `y2`: random within video dimensions (use ffprobe)

Get video dimensions and duration via `ffprobe` before building the command.
Run watermark asynchronously with `asyncio.create_subprocess_exec` (never block).
